Janet Language

Janet is a functional and imperative programming language. It runs on Windows, Linux, macOS, and should run on other systems with some porting. The entire language (core library, interpreter, compiler, assembler) is about 200-300 kB and should run on many constrained systems.

Features

  • Garbage Collection
  • Self contained compiler, interpreter, core library, and bytecode assembler
  • First class closures
  • First class green threads (continuations)
  • Mutable and immutable arrays (array/tuple)
  • Mutable and immutable hashtables (table/struct)
  • Mutable and immutable strings (buffer/string)
  • Lisp Macros
  • Direct interop with C via abstract types and C functions
  • Dynamically load C libraries
  • Functional and imperative standard library
  • Lexical scoping
  • REPL
  • Interactive environment with detailed stack traces

# Fizzbuzz
(defn fizz-buzz
  "Do fizz-buzz for 1-n"
  [n]
  (loop [i :range [1 n]]
    (print (cond
      (zero? (% i 15)) "fizz buzz"
      (zero? (% i 5)) "fizz"
      (zero? (% i 3)) "buzz"
      i))))

Usage

To use janet, you will need to build the project from source. The intended way to configure your build of janet is to modify the header file in the source code src/include/janet/janet.h. Standard configurable options are at the top of janet.h and are commented out by default.

Linux and macOS Steps

On Linux and macOS systems, Janet is built with a plain Makefile (No autotools, Cmake, gyp, premake, scons, etc).

  1. Clone the repository from GitHub on to your local machine.
  2. From the repository, run make to build the janet executable program.
  3. Run make test to verify everything is working correctly. All tests should pass and make test should return with an exit code of 0.
  4. To install globally, run sudo make install. You can change the install location by setting the PREFIX option when installing, such as sudo make PREFIX=/usr/install.

Windows Steps

On Windows, Janet can be built with Visual Studio or with the Windows Build tools. The Windows build script is a batch file build_win.bat that must be run from a Visual Studio developer console.

  1. As with Linux or macOS, clone the repository from GitHub on to your local machine.
  2. Open a Visual Studio Developer console on your local machine. This can usually be found under the Visual Studio folder if Visual Studio is installed on the machine.
  3. Run the command build_win in the console window to build the janet executable.
  4. Run the command build_win test to verify that everything is working.

Documentation
